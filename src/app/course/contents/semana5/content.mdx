# Programación Orientada a Objetos (POO) en Java:

**Clase**: Una plantilla utilizada para crear objetos. Contiene métodos y atributos que definen el comportamiento y las características de un objeto.

**Objeto**: Una instancia de una clase. Los objetos encapsulan datos (atributos) y métodos que pueden interactuar con esos datos.

**Atributo (Campo)**: Una variable perteneciente a una clase o objeto que representa una característica o propiedad del mismo.

**Método**: Una función que define un comportamiento o una acción que puede realizar un objeto.

**Constructor**: Un método especial que se utiliza para inicializar un objeto. Se ejecuta cuando se crea una nueva instancia de una clase.

**Herencia**: Un mecanismo que permite a una clase heredar atributos y métodos de otra clase (superclase). La clase que hereda se denomina subclase.

**Superclase (Clase base o Clase padre)**: Una clase que es heredada por una o más subclases.

**Subclase (Clase derivada o Clase hija)**: Una clase que hereda atributos y métodos de una superclase.

**Sobreescritura de métodos**: El proceso de definir un método en una subclase que ya está definido en la superclase. Permite modificar o ampliar la funcionalidad del método original.

**Polimorfismo**: La capacidad de un objeto de ser interpretado de forma diferente en función de la situación. En Java, se logra principalmente mediante la sobreescritura de métodos y la implementación de interfaces.

**Encapsulamiento**: La práctica de agrupar atributos y métodos en una clase para protegerlos y restringir su acceso desde fuera de la clase.

**Información privada**: Los atributos y métodos que solo pueden ser accedidos dentro de la propia clase. En Java, se marcan con el modificador private.

**Información pública**: Los atributos y métodos que pueden ser accedidos desde dentro y fuera de la clase. En Java, se marcan con el modificador public.

**Modificadores de acceso**: Palabras clave utilizadas para especificar el nivel de acceso a atributos y métodos en Java. Incluyen public, private, protected y default (sin modificador explícito).

**Interface**: Una colección de métodos abstractos que establecen un contrato que deben implementar las clases que la implementan.

**Abstracción**: La práctica de ocultar la implementación interna de un objeto y solo mostrar las funciones que pueden realizar.

**Clase abstracta**: Una clase que no puede ser instanciada y que está diseñada solo para ser extendida por otras clases. En Java, se define con la palabra clave abstract.
