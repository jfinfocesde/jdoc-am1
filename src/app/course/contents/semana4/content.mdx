# Constructores en Java

Un constructor es un método especial que se ejecuta al crear una nueva instancia de una clase en Java. Su función principal es inicializar los atributos del objeto recién creado.

**Características de los constructores:**

-   Tienen el mismo nombre que la clase.
-   No tienen un tipo de retorno.
-   Se pueden definir con diferentes parámetros.
-   Si no se define ningún constructor, Java crea un constructor por defecto sin parámetros.
-   Se pueden sobrecargar, es decir, tener dos o más constructores con diferentes parámetros.

### **Ejemplo:**

```java
public class Persona {
  private String nombre;
  private int edad;

  public Persona() {
    // Constructor por defecto
  }

  public Persona(String nombre, int edad) {
    // Constructor con parámetros
    this.nombre = nombre;
    this.edad = edad;
  }
}
```

En este ejemplo, la clase `Persona` tiene dos constructores:

-   **Constructor por defecto:** No tiene parámetros y se encarga de inicializar los atributos `nombre` y `edad` con valores por defecto ("" para `nombre` y 0 para `edad`).
-   **Constructor con parámetros:** Recibe como parámetros el nombre y la edad de la persona, y se encarga de inicializar los atributos correspondientes.

### **Uso de los constructores:**

Los constructores se utilizan para:

-   **Inicializar los atributos de un objeto.**
-   **Realizar validaciones sobre los valores de los atributos.**
-   **Ejecutar código específico al crear una nueva instancia de una clase.**

### **Ejemplo de uso:**

```java
Persona persona1 = new Persona(); // Se crea una persona con el constructor por defecto

Persona persona2 = new Persona("Juan", 30); // Se crea una persona con el constructor con parámetros

System.out.println("Nombre: " + persona1.getNombre()); // Imprime ""
System.out.println("Edad: " + persona1.getEdad()); // Imprime 0

System.out.println("Nombre: " + persona2.getNombre()); // Imprime "Juan"
System.out.println("Edad: " + persona2.getEdad()); // Imprime 30
```
# Herencia en Java:

La herencia en Java es un mecanismo que permite a una clase (subclase) heredar las características (atributos y métodos) de otra clase (superclase). Esta es una de las características clave de la programación orientada a objetos que facilita la reutilización de código y la organización del mismo.

La herencia en Java es una característica clave del paradigma de programación orientada a objetos (POO), que permite a las clases heredar propiedades y comportamientos de otras clases.

La palabra reservada extends se utiliza para establecer una relación de herencia entre dos clases. La clase que extiende se llama "subclase" o "clase derivada", mientras que la clase de la que se extiende se conoce como "superclase" o "clase base".

**Beneficios de la herencia:**

-   **Reutilización de código:** Permite evitar la duplicación de código al heredar características de una clase ya existente.
-   **Extensibilidad:** Permite crear nuevas clases a partir de clases existentes, extendiendo su funcionalidad.
-   **Mantenimiento:** Facilita el mantenimiento del código al centralizar la lógica en la superclase.
-   **Organización:** Ayuda a organizar el código en una jerarquía de clases.
### **Tipos de herencia en Java:**

-   **Herencia simple:** Una subclase hereda de una única superclase.
-   **Herencia jerárquica:** Una superclase tiene una o más subclases.
-   **Herencia multinivel:** Una subclase hereda de otra subclase, que a su vez hereda de otra, y así sucesivamente.
### **Ejemplo de herencia simple:**

```java
public class Persona {
  private String nombre;
  private int edad;

  public String getNombre() {
    return nombre;
  }

  public void setNombre(String nombre) {
    this.nombre = nombre;
  }

  public int getEdad() {
    return edad;
  }

  public void setEdad(int edad) {
    this.edad = edad;
  }
}

public class Estudiante extends Persona {
  private String matricula;

  public String getMatricula() {
    return matricula;
  }

  public void setMatricula(String matricula) {
    this.matricula = matricula;
  }
}
```

En este ejemplo, la clase `Estudiante` hereda de la clase `Persona`. Esto significa que:

-   La clase `Estudiante` tiene todos los atributos y métodos de la clase `Persona`.
-   Se pueden crear objetos de la clase `Estudiante` que tendrán nombre, edad y matrícula.
-   Se pueden utilizar los métodos `getNombre()`, `setNombre()`, `getEdad()` y `setEdad()` con objetos de la clase `Estudiante`.
### **Modificadores de acceso y herencia:**

Los modificadores de acceso controlan quién puede acceder a los atributos y métodos de una clase. En el contexto de la herencia, es importante tener en cuenta lo siguiente:

-   Los atributos y métodos con modificador `public` son accesibles desde cualquier clase.
-   Los atributos y métodos con modificador `protected` son accesibles desde la clase en la que se definen y desde sus subclases.
-   Los atributos y métodos con modificador `private` solo son accesibles desde la clase en la que se definen.
### **Ejemplo de modificadores de acceso y herencia:**

```java
public class Persona {
  public String nombre; // Public, accesible desde cualquier clase
  protected int edad; // Protected, accesible desde Persona y sus subclases
  private String telefono; // Private, solo accesible desde Persona
}

public class Estudiante extends Persona {
  public String getInformacion() {
    return "Nombre: " + nombre + " - Edad: " + edad; // Se puede acceder a "nombre" y "edad" porque son "public" y "protected"
    // return "Nombre: " + nombre + " - Edad: " + telefono; // No se puede acceder a "telefono" porque es "private"
  }
}
```

## Constructores y herencia en Java

En Java, los constructores no se heredan de la superclase a la subclase. Sin embargo, existen mecanismos para trabajar con constructores en el contexto de la herencia:

**1\. Llamada al constructor de la superclase:**

-   Se utiliza la palabra clave `super` para invocar al constructor de la superclase desde el constructor de la subclase.
-   Es obligatorio llamar al constructor de la superclase antes de realizar cualquier otra acción en el constructor de la subclase.
### **Ejemplo:**

```java
public class Persona {
  public Persona(String nombre) {
    this.nombre = nombre;
  }
}

public class Estudiante extends Persona {
  private String matricula;

  public Estudiante(String nombre, String matricula) {
    super(nombre); // Llama al constructor de la superclase con el nombre
    this.matricula = matricula;
  }
}
```

En este ejemplo, la clase `Estudiante` necesita el nombre y la matrícula para crear un nuevo objeto. Se llama al constructor de la superclase (`Persona`) con el nombre y luego se inicializa el atributo `matricula`.


## La palabra reservada super 

La palabra reservada super se utiliza dentro de la subclase para acceder a los métodos y atributos de su superclase. Aquí hay algunos usos comunes de la palabra reservada super:

1.  Invocar métodos de la superclase: Si la subclase sobrecarga un método de la superclase, puede acceder a la implementación del método de la superclase utilizando la palabra reservada super seguida del nombre del método. Por ejemplo:


```java
class Superclass {
  void method() {
    System.out.println("Superclass method");
  }
}

class Subclass extends Superclass {
  void method() {
    super.method(); // Invoca el método de la superclase
    System.out.println("Subclass method");
  }
}

public class Main {
  public static void main(String[] args) {
    Subclass obj = new Subclass();
    obj.method();
  }
}
```

2.  Acceder a los atributos de la superclase: Si la subclase necesita acceder a un atributo de la superclase, puede utilizar la palabra reservada super seguida del nombre del atributo.
3.  Invocar el constructor de la superclase: Si la subclase tiene un constructor y la superclase también, el constructor de la subclase debe invocar al constructor de la superclase mediante la palabra reservada super en la primera línea del constructor de la subclase.

```java
class Superclass {
  Superclass(String name) {
    System.out.println("Superclass constructor");
  }
}

class Subclass extends Superclass {
  Subclass(String name) {
    super(name); // Invoca el constructor de la superclase
    System.out.println("Subclass constructor");
  }
}

public class Main {
  public static void main(String[] args) {
    Subclass obj = new Subclass("example");
  }
}
```

La herencia en Java promueve la reutilización del código y la facilidad de mantenimiento de las aplicaciones. La palabra reservada super juega un papel crucial en la interacción entre clases base y derivadas al permitir a las clases derivadas acceder a los miembros de sus clases base.

## Ejemplo 

```java
class Empleado {
  private String nombre;
  private int idEmpleado;
  private double salario;

  public Empleado(String nombre, int idEmpleado, double salario) {
    this.nombre = nombre;
    this.idEmpleado = idEmpleado;
    this.salario = salario;
  }

  public String getNombre() {
    return nombre;
  }

  public int getIdEmpleado() {
    return idEmpleado;
  }

  public double getSalario() {
    return salario;
  }

  public void mostrarDetalles() {
    System.out.println("Nombre del empleado: " + nombre);
    System.out.println("ID del empleado: " + idEmpleado);
    System.out.println("Salario: " + salario);
  }
}

class Gerente extends Empleado {
  private String departamento;

  public Gerente(String nombre, int idEmpleado, double salario, String departamento) {
    super(nombre, idEmpleado, salario);
    this.departamento = departamento;
  }

  public String getDepartamento() {
    return departamento;
  }

  public void mostrarDetalles() {
    super.mostrarDetalles();
    System.out.println("Departamento: " + departamento);
  }
}

class Desarrollador extends Empleado {
  private String lenguajeProgramacion;

  public Desarrollador(String nombre, int idEmpleado, double salario, String lenguajeProgramacion) {
    super(nombre, idEmpleado, salario);
    this.lenguajeProgramacion = lenguajeProgramacion;
  }

  public String getLenguajeProgramacion() {
    return lenguajeProgramacion;
  }

  public void mostrarDetalles() {
    super.mostrarDetalles();
    System.out.println("Lenguaje de programación: " + lenguajeProgramacion);
  }
}

class Tecnico extends Empleado {
  private String areaDeEspecializacion;

  public Tecnico(String nombre, int idEmpleado, double salario, String areaDeEspecializacion) {
    super(nombre, idEmpleado, salario);
    this.areaDeEspecializacion = areaDeEspecializacion;
  }

  public String getAreaDeEspecializacion() {
    return areaDeEspecializacion;
  }

  public void mostrarDetalles() {
    super.mostrarDetalles();
    System.out.println("Área de especialización: " + areaDeEspecializacion);
  }
}

public class Main {
  public static void main(String[] args) {
    Empleado emp1 = new Empleado("Juan", 1, 50000);
    Gerente mgr1 = new Gerente("María", 2, 80000, "Ventas");
    Desarrollador dev1 = new Desarrollador("Roberto", 3, 70000, "Java");
    Tecnico tec1 = new Tecnico("Lucas", 4, 60000, "Redes");

    emp1.mostrarDetalles();
    System.out.println("\n");
    mgr1.mostrarDetalles();
    System.out.println("\n");
    dev1.mostrarDetalles();
    System.out.println("\n");
    tec1.mostrarDetalles();
  }
}
```
